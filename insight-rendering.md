# 会后复盘渲染规范

> **加载契约**：进入阶段 3 会后复盘 Phase A 前，AI 必须 Read 本文件。

---

## Phase C — 智能渲染（纯 HTML/CSS 零外部 JS）

### 报告结构（根据会议类型动态组装）

```
[Hero 信息卡片] → [核心结论] → [关键决策] → [行动项] → [会议总览] → [时间线] → [场景洞察] → [原文引用]
```

**动态章节选择规则**（数据量决定，不强行填充）：

| 章节 | 触发条件 |
|------|---------|
| Hero 信息卡片 | 始终需要 |
| 核心结论 | 结论 ≥ 3 条（1-2 条放在 Hero 底部摘要区） |
| 关键决策 | 决策 ≥ 3 条（1-2 条并入结论区） |
| 行动项 | 行动项 ≥ 1 |
| 会议总览 | 主题 ≥ 4 个（<4 个主题不值得可视化） |
| 时间线 | 时间节点 ≥ 3 个（2 个节点不需要"时间线"） |
| 对比分析 | 有明确对比内容（如与上期会议对比） |
| SWOT | 类型为战略/复盘/客户拜访 |
| 健康度评分卡 | 类型为客户拜访/客户成功 |
| 决策质量评分 | 类型为决策复盘 |
| 沟通盲点分析 | 类型为沟通复盘 |
| 跨学科洞见 | 类型为创意复盘 |
| 效率评分卡 | 完整度 ≥ 中 |
| 原文引用支撑 | 完整度 ≥ 中 |

**密度模式**：

| 模式 | 最大章节数 | 触发 |
|------|-----------|------|
| compact | 4（Hero + 2 核心 + 行动） | 低信息量 或 站会/1:1 |
| normal | 8 | 默认 |
| rich | 12 | 高信息量 或 战略/复盘/客户会议 |

**信息图节奏**：连续信息图不超过 3 张。第 4 个起交替表格或结构化文本。

## 诚实分层标注（必须严格遵守）

| 标注 | 含义 | 使用场景 |
|------|------|---------|
| `[原文引用]` | 直接引用，未做任何语言转换 | 逐字稿中完整、清晰的句子 |
| `[转写]` | 口语转为书面语，保留原意和完整事实 | 去除填充词、修正语法后的版本 |
| `[概括]` | 多句话合并为一条结构化信息 | 一段讨论浓缩为一句话结论 |
| `[推断]` | 从语境间接推导，原始话语中未明确表达 | 必须标注置信度 |

**规则**：
- 尽力使用 `[原文引用]` 和 `[转写]`
- 谨慎使用 `[概括]`
- `[推断]` 必须标注置信度
- **不得**将口语重构伪装为原文引用
- 缺失数据明确标记

## 品牌规范

| 元素 | 值 |
|------|---|
| 品牌色 | `#0066FF` |
| 文字色 | `#1f2937` |
| 背景色 | `#f6f9fc` |
| 圆角 | `12px` |
| 最大宽度 | 800px |
| 字体 | `-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif` |

### CSS 禁止模式

```css
/* ❌ 禁止：直角色条 + 圆角（产生视觉裂隙）*/
border-left: 4px solid #0066FF;
border-radius: 12px;

/* ✅ 替代：顶边色条 */
border-top: 3px solid #0066FF;
border-radius: 12px;

/* ✅ 或：微妙阴影 */
box-shadow: 0 2px 8px rgba(0,102,255,.08);
```

### 可访问性

- HTML 报告 `<html lang="zh-CN">"`
- 图表给出文字摘要
- 颜色不作为唯一信息载体

## 密度诚实原则

**信息完整度越低，视觉越朴素。**

| 完整度 | 视觉策略 |
|--------|---------|
| 高（含说话人+时间戳） | 全部视觉品质释放，信息图丰富 |
| 中（逻辑连贯段落） | 标准视觉，推定项显式标注 |
| 低（三五句话） | **降级视觉精致度**：减信息图数量，增大标注字号和对比度 |

**禁止**：用精美的信息图包装空洞的内容。

## 场景洞察可视化组件

| 会议类型 | 洞察维度 | 可视化组件 |
|---------|---------|-----------|
| 站会 | 阻塞项 + 风险升级 | 阻塞清单 + 风险信号 |
| 周会 | 议题覆盖 + 决策效率 | 议题完成率 + 决策产出 |
| 复盘会 | 对比趋势 + 效率评分 | 评分卡 + 对比表格 |
| 启动会 | 目标对齐 + 责任矩阵 | 时间线 + 责任分配表 |
| 一对一 | 反馈质量 + 跟进 | 反馈评分卡 + 行动清单 |
| 战略会 | SWOT + 资源分配 | SWOT 象限 + 里程碑时间线 |
| 客户会议 | 客户诉求 + 异议 + 成交信号 | 诉求清单 + 状态流转 |
| 跟进会 | 行动项完成率 + 客户状态 | 进度条 + 状态卡片 |
| 客户拜访 | 五维健康度（使用深度/满意度/关系稳定性/续约意愿/增购潜力）| 雷达图 + 趋势对比 + 风险信号列表 + 增购机会 |
| 决策复盘 | 决策类型（快速/深思/犹豫/推迟）+ 决策依据（数据/直觉/他人/风险）+ 质量评分 | 决策树 + 依据分布 + 盲区识别 + 改进建议 |
| 沟通复盘 | 七类盲点（表达模糊/打断/话题跳跃/过度解释/情绪化/被动回应/绝对化）| 频次分布 + 场景差异 + 改进目标 + 追踪对比 |
| 创意复盘 | 跨学科视角（心理学/哲学/社会学等）| 学科卡片 + 意外洞见 + 故事关联 + 创作方向 |
| 投资者关系 | Q&A 分析 + 技术展望 + 投资者反馈 | 问答分类 + 关注热度 + 后续跟进 |
| 对齐会 | 依赖项 + 责任边界 | 依赖清单 + 风险矩阵 |
| OKR 复盘 | 完成度 + 偏差归因 | 完成度雷达 + 调整建议 |
| 绩效面谈 | 目标差距 + 改进计划 | 差距分析 + 能力评估表 |
| 通用会议 | 效率评分（议程达成率/决策产出率/参与均衡度/行动明确度）| 评分卡 + 改进建议 |

> **默认规则**：未明确列出的会议类型统一使用「通用会议」的评分卡 + 改进建议模式。

## 报告输出

- 输出路径：`docs/.tmp/复盘报告-{meeting_id}-{日期}.html`（该目录已加入 `.gitignore`，防止误提交）
- 输出后 `start ""` 打开浏览器预览
- 主动问："需要把复盘报告通过邮件发给参会人吗？"
- 如果是系列会议（如每周例会）：主动建议"需要和上次同类会议做趋势对比吗？"

## 紧凑模式空状态

当完整度 = 低且数据不足以生成标准 Hero 卡片时：
- 渲染一句说明："可用信息有限，以下基于现有数据的分析，标注 `[推断]` 的项目置信度较低。"
- 最小骨架：Hero 精简版（仅主题+时间）+ 核心结论 + 行动项
