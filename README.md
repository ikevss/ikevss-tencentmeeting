# 腾讯会议·邀约洞察

> 从建会到复盘，一个对话框走完全程。

### 你的痛点

- **建会麻烦**：打开客户端、填表单、复制链接、发通知——一场会前要花 15 分钟
- **通知建完就完**：发完邮件没人追踪，开会时才发现关键人没到
- **复盘成本高**：听录音、扒纪要、做 PPT——一场 1 小时的会复盘要花 3 小时
- **企业账号用不了**：腾讯官方 MCP 方案需要手动获取 Token，企业账号拿不到

**这个技能把四个环节串成一条链路**，你对 AI 说话，它帮你跑完全程。

---

## 解决什么问题

| 环节 | 没有这个技能 | 有了这个技能 |
|------|------------|------------|
| 建会 | 打开客户端填表单，5 分钟 | "帮我建一个明天下午3点的会"，10 秒 |
| 通知 | 手动复制链接发邮件，容易漏人 | 自动生成精美邀请函 + 日历附件，一键发送 |
| 复盘 | 听录音扒纪要，半天 | "帮我复盘上周的会"，生成可视化报告 |
| 企业账号 | 官方 MCP 用不了 | OAuth2 浏览器授权，个人企业都能用 |

## 什么时候会触发

你说以下任何一句话，AI 就会启动对应环节：

**建会**
- "帮我建一个明天下午3点的腾讯会议"
- "创建一个每周一早上的周会，到年底"

**发通知**
- "给参会人发邮件邀请"
- "把邀请函发给 zhang@company.com 和 li@company.com"

**会后复盘**
- "帮我复盘上周的产品评审会"
- "分析一下这场客户拜访，关注满意度和后续意向"
- "追踪上个月战略会上所有决议的执行情况"

**其他**
- "帮我登录腾讯会议"（首次授权）
- "查一下上次路演的录制和纪要"
- "搜通讯录里投资者关系部门的人"

## 四个环节，一句话串联

```
你说"帮我建一个明天下午3点的会，讨论Q2财报，通知张三和李四"
  │
  ├─ ① 建会 → 拿到会议号 + 入会链接
  │
  ├─ ② 生成邀请函 → 浏览器预览确认
  │
  ├─ ③ 发送邮件 → 精美 HTML + 日历附件（双击加进日程）
  │
  └─ ④ 会后复盘 → 拉取录制转写 → 生成可视化报告
```

## 能做什么

### 📅 建会
- 一句话创建，支持周期会议、成员管理、密码设置
- 自动检测时间冲突，有重叠会提醒你

### 📧 发通知
- 自动生成精美 HTML 邀请函（手机电脑都能看）
- 附带 .ics 日历文件，收件人双击即可加进日程
- 发送前浏览器预览，确认后再发

### 📊 复盘洞察（核心差异化）

不只是"会议纪要"，而是**带方法论的深度复盘**。

**诚实分层标注**（防 AI 幻觉）：
- `[原文引用]` > `[转写]` > `[概括]` > `[推断]`
- 每段信息标注来源层级，推断必须标置信度

**密度诚实原则**：
- 信息完整度低时，视觉自动降级——不用精美图表包装空洞内容

**18 种会议类型 × 专属分析视角**：

| 类型 | 核心洞察 | 可视化 |
|------|---------|--------|
| 客户拜访 | 五维健康度（使用深度/满意度/关系/续约/增购）| 雷达图 + 趋势对比 |
| 决策复盘 | 决策类型（快速/深思/犹豫/推迟）+ 依据 + 质量评分 | 决策树 + 盲区识别 |
| 沟通复盘 | 七类盲点（表达模糊/打断/话题跳跃/过度解释/情绪化/被动/绝对化）| 频次分布 + 改进追踪 |
| 创意复盘 | 跨学科视角（心理学/哲学/社会学等）| 学科卡片 + 创作方向 |
| 投资者关系 | Q&A 分类 + 技术展望 + 投资者反馈 | 问答热度 + 跟进清单 |
| OKR 复盘 | 完成度雷达 + 偏差归因 + 资源瓶颈 | 完成度趋势 + 调整建议 |
| 绩效面谈 | 目标差距 + 能力评估 + 员工意愿信号 | 差距分析 + 改进计划 |

**跨会议追踪**：
- 决策执行闭环：本期决策 → 下期自动回顾执行情况
- 趋势对比：同类会议多次召开，自动追踪决策质量/健康度/效率变化
- 会议 ROI：成本（人时×时长）vs 产出（决策数×权重）

**参与者贡献分析**：
- 发言时长分布 + 关键贡献者识别 + 沉默者标记

**输出**：可视化 HTML 报告（浏览器打开），可邮件发送给参会人

### 🔐 企业账号通用
- OAuth2 浏览器授权，不区分个人/企业
- 凭证 AES-256-GCM 加密存储

## 快速开始

### 1. 安装

```bash
# 方式一：一键安装（推荐）
node install.js

# 方式二：手动安装
npm install -g @tencentcloud/tmeet@v1.0.12
cp SKILL.md ~/.claude/skills/
```

安装后 **重启 Claude Code**。

### 2. 登录（30 秒，只需一次）

```
帮我登录腾讯会议
```

浏览器自动弹出，登录并授权。之后永久免登录。

### 3. 开始使用

```
帮我创建一个腾讯会议，明天下午3点讨论Q2财报，时长1小时
```

## 常见问题

<details>
<summary><b>企业账号能用吗？</b></summary>

能。使用 OAuth2 浏览器授权，腾讯会议不区分个人/企业，只要账号能登录就能授权。
</details>

<details>
<summary><b>登录超时怎么办？</b></summary>

不要加 `--no-browser` 参数。让 CLI 自动弹出浏览器窗口，浏览器中有完整的登录和授权流程，不容易超时。
</details>

<details>
<summary><b>发邮件需要额外装什么？</b></summary>

需要 agently-cli（发邮件）和 react-email（生成精美邮件）。创建会议后如果选择发邀请，AI 会自动检查并引导安装。
</details>

<details>
<summary><b>如何卸载？</b></summary>

```bash
npm uninstall -g @tencentcloud/tmeet
rm ~/.claude/skills/SKILL.md
```
</details>
