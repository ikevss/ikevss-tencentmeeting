## Phase B — 章节选择边界 Case

### Case 1：信息量极少（低密度）

**输入**：用户只说"帮我复盘上周的会"，无转写原文，只有 3 句话描述。

**处理**：
- 完整度 = 低 → 降级视觉精致度
- 章节数 = compact（4 章）：Hero + 核心结论 + 行动项 + 原文引用
- 推定项显式标注 `[推断]`

### Case 2：信息量极大（高密度）

**输入**：2 小时董事会完整逐字稿，8 个发言人，15 个主题。

**处理**：
- 完整度 = 高 → 全部视觉品质释放
- 章节数 = rich（12 章）：全部章节启用
- 信息图节奏控制：连续不超过 3 张，第 4 张起交替表格

### Case 3：模糊时间指示

**输入**：用户说"帮我复盘上次路演"，但一个月内路演了 5 次。

**处理**：触发多会议消歧义 → 列出候选会议（主题·时间·参会人数）→ 让用户选编号

## 15 种会议类型详细分析视角

### 1. Daily Standup（站会）
- **触发词**：站会/sync/每日/morning
- **布局**：compact（最多 4 章）
- **洞察维度**：阻塞项识别 + 昨日完成 vs 今日计划 + 风险升级

### 2. Weekly Sync（周会）
- **触发词**：周会/weekly/双周
- **布局**：normal + 章节详解
- **洞察维度**：议题覆盖度 + 决策效率 + 行动项闭环

### 3. Retrospective（复盘会）
- **触发词**：复盘/retro/review/评估
- **布局**：含对比分析 + 评分布局 + 效率评分卡
- **洞察维度**：做得好的 / 需改进的 / 下次行动 + 与上期对比趋势

### 4. Kickoff（启动会）
- **触发词**：启动/kickoff/计划/plan
- **布局**：含时间线 + 责任矩阵
- **洞察维度**：目标对齐 + 里程碑设定 + 角色分工 + 风险预判

### 5. One-on-One（一对一）
- **触发词**：1:1/one-on-one/双人
- **布局**：精简，无双人以上信息图
- **洞察维度**：双方议题覆盖 + 反馈质量 + 后续跟进

### 6. Strategic（战略会）
- **触发词**：战略/strategy/规划/半年/年度
- **布局**：rich + SWOT + 象限 + 效率评分卡
- **洞察维度**：SWOT 分析 + 战略决策 + 资源分配 + 里程碑

### 7. Client Meeting（客户会议）
- **触发词**：客户/路演/sales/pitch
- **布局**：商务浅色主题
- **洞察维度**：客户诉求 + 异议处理 + 下一步行动 + 成交信号

### 8. Follow-up（跟进会）
- **触发词**：跟进/follow-up/CRM/pipeline
- **布局**：行动导向：分布 + 时间线 + 客户状态
- **洞察维度**：上期行动项完成率 + 客户状态变化 + 下一步优先级

### 9. Investor Relations（投资者关系）
- **触发词**：路演/投资人/IR/投资者/募资
- **布局**：商务浅色
- **洞察维度**：Q&A 分类（业务/财务/技术/竞争）+ 投资者关注热度 + 后续跟进清单

### 10. Customer Success（客户成功）
- **触发词**：客户/回访/拜访/续约/增购/交付
- **布局**：含健康度评分卡
- **洞察维度**：五维健康度（使用深度/满意度/关系稳定性/续约意愿/增购潜力）+ 流失风险信号 + 增购机会 + 趋势对比

### 11. Decision Review（决策复盘）
- **触发词**：决策/评审/拍板/定方案/选型/通过
- **布局**：含决策质量评分
- **洞察维度**：决策类型识别（快速/深思/犹豫/推迟）+ 决策依据（数据/直觉/他人/风险）+ 质量评分（及时性/信息完备性/参与度/可执行性）+ 决策盲区 + 改进建议

### 12. Communication（沟通复盘）
- **触发词**：一对一/1on1/沟通/反馈/绩效/辅导
- **布局**：含沟通盲点分析
- **洞察维度**：七类盲点识别（表达模糊/打断/话题跳跃/过度解释/情绪化/被动回应/绝对化）+ 频次分布 + 场景差异 + 改进目标 + 追踪对比

### 13. Creative（创意复盘）
- **触发词**：脑暴/讨论/发散/创意/方案/探索
- **布局**：含跨学科洞见
- **洞察维度**：跨学科视角（心理学/哲学/社会学/历史/科学）+ 意外洞见 + 故事关联 + 创作方向 + 新方案

### 14. Alignment（对齐会）🆕
- **触发词**：对齐/同步/例会/里程碑/跨部门
- **布局**：normal + 依赖项清单
- **洞察维度**：依赖项清单 + 责任边界 + 风险升级点 + 跨团队承诺

### 15. OKR Review（OKR 复盘）🆕
- **触发词**：OKR/KR/季度/完成率/偏差
- **布局**：含完成度雷达
- **洞察维度**：完成度雷达（O1/O2/O3...）+ 偏差归因 + 下周期调整建议 + 资源瓶颈识别

### 16. Performance Review（绩效面谈）🆕
- **触发词**：绩效/考核/晋升/转正/面谈
- **布局**：含目标差距分析
- **洞察维度**：目标差距（KPI 完成率）+ 能力评估（硬技能/软技能）+ 改进计划 + 员工意愿信号 + 发展建议

### 17. General（通用会议）
- **触发词**：无特定关键词
- **布局**：标准布局
- **洞察维度**：效率评分（议程达成率/决策产出率/参与均衡度/行动明确度）+ 改进建议

## 口语语言学预处理（第〇步，不得跳过）

### 口语 → 结构化转换

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1. 去除填充 | 删除填充词 | "那个、就是说、嗯、然后然后" → 删除 |
| 2. 消解指代 | 将"那个"、"这边"、"他"替换为明确实体 | "那个你跟进一下" → "供应商付款你跟进一下" |
| 3. 合并修正 | 将自我修正合并为最终版本 | "下周一——不，周三开会" → "周三开会" |
| 4. 保留情态 | 保留说话人的不确定性标记 | "我觉得"、"可能"、"应该" → 保留，作为可靠性信号 |

### 语言转换层级

| 标注 | 含义 | 使用场景 |
|------|------|---------|
| `[原文]` | 直接引用，未做任何语言转换 | 逐字稿中完整、清晰的句子 |
| `[转写]` | 口语转为书面语，保留原意和完整事实 | 去除填充词、修正语法后的版本 |
| `[概括]` | 多句话合并为一条结构化信息 | 一段讨论浓缩为一句话结论 |
| `[推断]` | 从语境间接推导，原始话语中未明确表达 | 必须标注置信度 |

**规则**：尽力使用 `[原文]` 和 `[转写]`，谨慎使用 `[概括]`，`[推断]` 必须标注置信度。
